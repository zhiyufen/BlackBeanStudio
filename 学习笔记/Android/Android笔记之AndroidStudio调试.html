2， 条件断点

  当你只在满足某些条件下时才需要断点，就可能使用这个；
  使用方法：在你的断点上鼠标右键会出现一个小窗口，写上条件即可。

3, 日志断点

  日志断点并不会打断当前运行，只是打印当前环境某些变量的Log（但不需要在代码中添加打印Log），这样就可以防止重新编译APK等流程。
  使用方法：在你的断点上鼠标右键会出现一个小窗口，Disable掉Suspend选项，这样就显示下面的界面，然后填写需要打印的信息。

4, 方法断点

  当你在一个方法行打上断点时，除了会在运行到当前方法会进行断点，在方法退回时，也会进行断点，这样我们就可以查看方法传入参数和最后的运行结果。

5, 异常断点

  当前调试应用发生异常时，Android Studio能进行断点处理，这个就有更多信息进行分析程序当前各变量的情况。
  使用方法： Run -> View BreakPoints 或者使用快捷键打开断点设置窗口。
